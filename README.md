# 🔍 Универсальный классификатор организаций

Инструмент для автоматической классификации списка компаний/организаций по любой теме. Использует комбинацию ключевых слов и языковой модели (Zero-Shot ML) для определения, подходит ли организация под заданные критерии.

## Как это работает

Классификация проходит в **три этапа** для каждой организации:

```
Название организации
        │
        ▼
┌─────────────────────┐
│  1. Сильные маркеры │  → Нашли совпадение → ✅ Целевой агент (уверенность 0.95)
└─────────────────────┘
        │ нет совпадений
        ▼
┌──────────────────────────┐
│  2. Исключающие маркеры  │  → Нашли совпадение → ❌ Нецелевой агент (уверенность 0.85)
└──────────────────────────┘
        │ нет совпадений
        ▼
┌────────────────────────────┐
│  3. Zero-Shot ML (модель)  │  → Классификация с вероятностью
└────────────────────────────┘
        │
        ▼
  Итоговая метка + уверенность
```

Средние маркеры не дают итоговой метки сами по себе, но повышают уверенность модели на +0.15, если ML-результат положительный.

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install pandas transformers tqdm openpyxl torch
```

### 2. Подготовьте входной файл

Создайте файл `companies.xlsx` с колонкой `company_name` (или другим именем — см. [настройки](#параметры-и-настройки)):

| company_name |
|---|
| ООО Ромашка |
| Фонд помощи детям |
| ... |

### 3. Настройте классификатор

Откройте `classifier_template.py` и заполните все места, помеченные комментарием `# ←`. Подробнее — в разделе [Настройка под свою задачу](#настройка-под-свою-задачу).

### 4. Запустите

```bash
python classifier_template.py
```

Результат появится в файле `companies_labeled.xlsx`.

## Настройка под свою задачу

Все настройки сосредоточены в верхней части файла. Ниже — описание каждого блока.

### 📁 Файлы и колонка

```python
INPUT_FILE = "companies.xlsx"       # Путь к вашему Excel-файлу
OUTPUT_FILE = "companies_labeled.xlsx"  # Куда сохранить результат
COMPANY_COLUMN = "company_name"     # Имя колонки с названиями организаций
```

### 🟢 Сильные маркеры (`STRONG_MARKERS`)

Слова, которые **однозначно** указывают на целевую организацию. При совпадении организация сразу получает метку «Целевой агент» с уверенностью 0.95 — без обращения к модели.

```python
STRONG_MARKERS = [
    "медицин",   # Охватит: медицина, медицинский, медицинская...
    "клиник",    # Охватит: клиника, клинический...
    "больниц",
]
```

> **Совет:** Используйте корень слова без окончания, чтобы охватить все словоформы. Регистр не важен.

### 🟡 Средние маркеры (`MEDIUM_MARKERS`)

Слова, которые *косвенно* указывают на тему. Повышают уверенность ML-классификации на +0.15, если модель уже склонна к положительному результату.

```python
MEDIUM_MARKERS = [
    "здоровь",
    "реабилит",
    "пациент",
]
```

### 🔴 Исключающие маркеры (`EXCLUDE_MARKERS`)

Слова, которые однозначно указывают, что организация **не подходит**. При совпадении (и отсутствии средних маркеров) организация сразу получает метку «Нецелевой агент».

```python
EXCLUDE_MARKERS = [
    "строи",
    "торг",
    "банк",
]
```

### 🏷️ Метки классификации (`ZS_LABELS`)

Описания для языковой модели на **английском языке**. Именно по ним модель решает, к какой категории отнести организацию.

Доступны три варианта разметки:

| Вариант | Когда использовать |
|---|---|
| `ZS_LABELS_V1` | **Стандарт.** Двухуровневый: «да / нет». Подходит для большинства задач. |
| `ZS_LABELS_V2` | Трёхуровневый: добавляет промежуточную категорию. Точнее, но сложнее. |
| `ZS_LABELS_V3` | Самый простой. Используйте, если V1 даёт плохие результаты. |

Активный вариант выбирается здесь:

```python
ZS_LABELS = ZS_LABELS_V1  # ← Замените на V2 или V3 при необходимости
```

Пример заполнения для поиска медицинских организаций:

```python
ZS_LABELS_V1 = [
    "Organization engaged in healthcare, medicine, patient treatment, or medical services",
    "Organization NOT engaged in healthcare or medical activities"
]
```

### 💬 Подсказка для модели (`build_prompt`)

Текст, который отправляется в модель вместе с названием организации. Чем точнее описание — тем лучше результат.

```python
def build_prompt(clean_name: str) -> str:
    return (
        f'Organization name: "{clean_name}". '
        f'Task: Determine if this organization provides medical services, '
        f'treats patients, or operates in the healthcare sector. '
        f'Administrative health agencies that do NOT treat patients '
        f'should be classified as NOT engaged.'
    )
```

### 🏷️ Названия итоговых меток

Что будет записано в результирующий Excel-файл:

```python
LABEL_POSITIVE  = "Целевой агент"     # Подходящие организации
LABEL_NEGATIVE  = "Нецелевой агент"   # Неподходящие организации
LABEL_UNCERTAIN = "Не определено"     # Низкая уверенность модели
```

## Входные данные

Файл `companies.xlsx` должен содержать хотя бы одну колонку с названиями организаций. Название колонки задаётся в `COMPANY_COLUMN`.

Дополнительные колонки (ИНН, адрес и т.д.) не помешают — они просто не будут использованы при классификации.

---

## Выходные данные

Файл `companies_labeled.xlsx` содержит следующие колонки:

| Колонка | Описание |
|---|---|
| `company_name` | Исходное название из входного файла |
| `clean_name` | Очищенное название (нижний регистр, без кавычек) |
| `label` | Итоговая метка классификации |
| `confidence` | Уверенность модели от 0.0 до 1.0 |
| `confidence_bucket` | Уровень уверенности: High / Medium / Low |
| `source` | Источник метки: Strong keyword / Exclude keyword / Zero-shot ML |
| `reasoning` | Краткое объяснение решения |

### Возможные значения `label`

| Метка | Значение |
|---|---|
| Целевой агент | Организация соответствует критериям |
| Нецелевой агент | Организация не соответствует критериям |
| Не определено | Модель не уверена (confidence < 0.50) |
| Invalid | Название слишком короткое или пустое |
| Error | Ошибка при обработке |

## Параметры и настройки

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `HIGH_CONF` | 0.70 | Порог для уровня «High» уверенности |
| `MEDIUM_CONF` | 0.50 | Порог для уровня «Medium»; ниже → метка «Не определено» |
| `CHECKPOINT_INTERVAL` | 50 | Каждые N компаний сохраняется промежуточный файл |

### Используемая модель

`MoritzLaurer/mDeBERTa-v3-base-mnli-xnli` — многоязычная модель для Zero-Shot классификации. Поддерживает русский язык. Загружается автоматически при первом запуске (~1.5 ГБ).

При запуске на CPU обработка одной организации занимает ~1–3 секунды. Для ускорения можно использовать GPU (`device=0` в коде).

## Часто задаваемые вопросы

**Почему метки нужно писать на английском?**
Модель обучена на английских NLI-датасетах и показывает лучшее качество при английских метках, даже если названия организаций на русском.

**Много результатов «Не определено» — что делать?**
Попробуйте: уточнить описание в `build_prompt`, добавить сильных маркеров в `STRONG_MARKERS`, или переключиться на `ZS_LABELS_V2` / `V3`.

**Можно ли использовать другой входной файл (не Excel)?**
Да — замените `pd.read_excel(INPUT_FILE)` на `pd.read_csv(INPUT_FILE)` и укажите `.csv`-файл.

**Прерванный запуск — данные потеряны?**
Нет. При прерывании через Ctrl+C сохраняется аварийный файл `emergency_ДАТА_ВРЕМЯ.xlsx` с уже обработанными организациями. Плюс каждые 50 компаний создаётся `checkpoint.xlsx`.
